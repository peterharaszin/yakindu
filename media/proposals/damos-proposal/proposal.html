The Damos project is a proposed open source project under the Model Development Tools (MDT) project.

This proposal is in the Project Proposal Phase (as defined in the Eclipse Development Process) and is written to declare its intent and scope. We solicit additional participation and input from the Eclipse community. Please send all feedback to the Eclipse Proposals Forum.

Background

In the field of science and engineering, data flow-oriented modeling is commonly used for analyzing and designing systems. On the one hand it allows scientists and engineers to focus on the data processing aspects of a system, and on the other hand it provides a means to specify the dynamics of the mechanical components and the physical environment of the system to facilitate close-to-real-world system simulation.

Since many scientists and engineers already utilize Eclipse for software development using CDT, DSDP, and other technologies, it would be beneficial to perform system modeling, simulation, and code generation in Eclipse rather than in an external tool.

Scope

This project will provide a data flow-oriented modeling environment. Furthermore, it will provide support to derive own data flow-oriented languages based on the structural and diagram editing components of this project.

Description

The goal of this project is to provide a modeling environment for analyzing and designing systems using a data flow-oriented approach. Such system models are usually comprised of software components (e.g. control algorithm), mechanical components, and the physical environment. Even though data flow-oriented design is not bound to a specific domain, it is most commonly used in control systems engineering and digital signal processing.

Architecture

In order to facilitate customization, Damos is split into a set of top-level components (i.e. Eclipse features). There are two main groups of components. The first group contains components providing the structural semantics and the diagram editing features of the modeling environment (orange). The second group provides simulation and code generation components as well as the standard block library (green). This enables other projects to build their own data flow-oriented languages by reusing the structural semantics and diagram editing components, and without being required to ship unneeded components.

Main Components

Models

EMF-based models are used to specify the semantics of data flow-oriented systems. This enables users to perform model analysis and transformation using well-known Eclipse modeling technologies.

System models are comprised of models describing the mechanical components and the physical environment of a device, and the software model that will later be executed on that device. These models are described graphically using block diagrams.

When performing model simulation or code generation, an execution flow is first derived from the system model. The execution flow contains the execution graphs (i.e. the execution order of system components) of the system model. The execution-specific configuration must be provided in an execution model. The simulation and the code generation models are used to configure the simulation process (e.g. solver configuration) and the code generator (e.g. target directories), respectively. The computation model specifies the computation algorithms utilized in the target device (e.g. fixed-point arithmetic vs. floating-point arithmetic).

Damos provides a GMF-based graphical block diagram editor for the system model. For all other models, textual model editors based on Xtext are provided.

Block Diagram Editor

The graphical block diagram editor is used for editing system models. It can also be used to visualize real-time simulation results using virtual instruments.

Simulation Environment

The simulation environment utilizes adaptive and fixed step size solvers to simulate continuous systems. It also provides asynchronous and real-time simulation capabilities.

Code Generator

The code generation tooling initially includes a standard compliant C code generator.

Block Library

The standard block library contains commonly used blocks. If required, the user is able to define his/her own blocks. This can be done using the declarative programming language Mscript, which is part of this project.

Extension Mechanisms

Extension mechanisms can be used to customize and extend various aspects of the tooling.
