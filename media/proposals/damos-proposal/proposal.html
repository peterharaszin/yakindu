The Damos project is a proposed open source project under the Model Development Tools (MDT) project.

This proposal is in the Project Proposal Phase (as defined in the Eclipse Development Process) and is written to declare its intent and scope. We solicit additional participation and input from the Eclipse community. Please send all feedback to the Eclipse Proposals Forum.

Background

Nowadays companies in the field of engineering are more and more struggling with the increasing software complexity of their systems. Engineers who were used to focus merely on physical and technical aspects of a system now need to deal with software engineering issues. The time pressure due to time-to-market constraints and the fact that many engineers are not adequately trained to handle such complex software structures lead to poor software quality, which results in less maintainable code—and more importantly—in an increase of system malfunctions.

To get a grip on the software complexity, a model-driven engineering approach becomes common practice. Since many engineers already utilize Eclipse for embedded software development using CDT, DSDP, and other technologies, it would be beneficial to perform system modeling, simulation, and code generation in Eclipse rather than in an external tool.

In the field of engineering, data flow-oriented designs are commonly used. On the one hand it allows engineers to focus on the data processing aspects of a technical system, and on the other hand it provides a means to specify the physical environment of the system to facilitate system simulation.

Scope

This project will provide:
  • Ecore models for specifying the semantics of data flow-oriented systems. This enables users to perform model analysis and transformation using well-known Eclipse modeling technologies.
  • A graphical block diagram editor for editing system models. This editor will also be used to visualize real-time simulation results using virtual instruments.
  • A simulation environment that utilizes adaptive and fixed step size solvers to simulate continuous systems. It will also provide asynchronous and real-time simulation capabilities.
  • Code generation tooling, which will initially include a C code generator.
  • Standard block library containing commonly used blocks. If required, the user will be able to define his/her own blocks. This can be done using a declarative programming language, which will be part of this project.
  • Extension mechanisms to customize and extend various aspects of the tooling.

Description

The goal of this project is to provide a modeling environment for modeling systems using a data flow-oriented approach. Such system models are usually comprised of software components (e.g. control algorithm), mechanical components, and the physical environment. Even though data flow-oriented design is not bound to a specific domain, it is most commonly used in control systems engineering and digital signal processing.
